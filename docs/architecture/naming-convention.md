# Naming Convention

**Project:** Atlas  
**Portfolio:** WolfSec Labs  
**Status:** Draft

## Purpose

Consistent naming makes enterprise infrastructure easier to manage and understand.

Rather than assigning random hostnames, Atlas follows a simple convention based on each system's role.

---

## Domain Controllers

DC01

Future domain controllers would become:

DC02

DC03

---

## Application Servers

APP01

Future servers:

APP02

APP03

---

## Windows Workstations

WS01

WS02

WS03

---

## Linux Systems

LINUX01

Future Linux servers will continue this sequence.

---

## Security Systems

KALI01

Future monitoring infrastructure will use names such as:

MON01

MON02

---

## Why this convention?

Role-based hostnames make troubleshooting, documentation and administration significantly easier as environments grow.

Someone reading "APP01" immediately understands that the machine provides application services without needing additional documentation.
